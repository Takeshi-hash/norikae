# 運用ルール：新規駅データのAI自律補完

このプロジェクトでは、ユーザーが「駅名のみ」で新規登録を行うことを許容しています。
Antigravity（私）は、新しい駅が追加されたことを検知した際、以下のルールに従って自律的にデータを補完します。

## 1. 補完のトリガー

- `routes.json` または Googleスプレッドシートに、`area` が「未分類（AI補完待ち）」または `yomi` が空の行を発見したとき。
- ユーザーから「運賃を更新して」「駅リストを確認して」などの依頼があった際の事前チェック。

## 2. 補完対象と基準

### a. よみがな (`yomi`)

- 到着駅名の正しい読みを「ひらがな」で入力する。
- ソート（50音順）の基準となるため、正確に入力すること。

### b. エリア (`area`)

以下のカテゴリから、駅の所在地や特徴に基づいて最も適切なものを選択する。

- **主要ターミナル**: 東京、品川、上野など
- **渋谷・新宿・池袋**: 渋谷、新宿、池袋、恵比寿など
- **港区（赤坂・六本木・麻布）**: 赤坂、六本木、神谷町、乃木坂、広尾、麻布十番など
- **千代田・中央オフィス街**: 大手町、日本橋、有楽町、新橋、虎ノ門、銀座など
- **中央線・文京エリア**: 飯田橋、市ヶ谷、四ツ谷、水道橋、本郷三丁目、護国寺など
- **湾岸エリア**: 豊洲、有明、台場、テレコムセンターなど
- **その他**: 上記に該当しないエリア

## 3. 実施手順

1. `routes.json` の該当エントリを書き換える。
2. スプレッドシートの該当行（A列：エリア、B列：よみ）を `update` する。
3. 補完が完了したら、ユーザーに「新しく追加された〇〇駅のデータを補完しておきました」と報告する。
